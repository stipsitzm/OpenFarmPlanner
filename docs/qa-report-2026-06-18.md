## Kritisch

### BUG-01 — Kein Bestätigungsdialog vor dem Löschen

**Bereich:** Anbaupläne, Anbauflächen (und vermutlich alle Module)

**Schritte zur Reproduktion:**
1. Anbaupläne öffnen (mobil oder Desktop)
2. Drei-Punkte-Menü / Rechtsklick auf eine Zeile öffnen
3. „Löschen" klicken

**Erwartet:** Bestätigungsdialog erscheint, z. B. „Möchtest du diesen Eintrag wirklich löschen?"

**Tatsächlich:** Eintrag wird sofort gelöscht. Kein Dialog, kein Toast, keine Rückgängig-Möglichkeit direkt am Lösch-Punkt.

**Hinweis:** Ein Versionsverlauf existiert unter Mehr → Versionsverlauf, ermöglicht aber nur projektweite Rollbacks, keine elementgenaue Wiederherstellung, und ist nicht mit der Lösch-Aktion verknüpft.

**Vorgeschlagene Lösung:** Bestätigungsdialog vor jeder Lösch-Aktion einbauen – oder zumindest einen „Gelöscht – Rückgängig"-Snackbar mit 8-Sekunden-Timeout anzeigen.

**Status:** ✅ Behoben (bereits vorhanden) – `2941e795` *„refactor: centralize confirmation prompts"* + `49942300` *„fix: focus safe button by default in confirmation dialogs"*

### BUG-02 — Rechtsklick „Bearbeiten" in Anbaupläne öffnet nichts (Desktop)

**Bereich:** Anbaupläne — Desktop

**Schritte zur Reproduktion:**
1. Anbaupläne bei Desktop-Viewport (≥ 1280 px) öffnen
2. Rechtsklick auf eine Zeile
3. „Bearbeiten" klicken

**Erwartet:** Bearbeiten-Modal öffnet sich (wie auf Mobil).

**Tatsächlich:** Zeile erhält einen Auswahlrahmen. Kein Modal, kein Inline-Edit. Die Aktion läuft ins Leere.

**Hinweis:** Auf Mobil funktioniert dieselbe Aktion korrekt und öffnet das „Anbauplan bearbeiten"-Modal.

**Vorgeschlagene Lösung:** Desktop-Rechtsklick-Handler so verbinden, dass er dasselbe Modal öffnet wie die mobile Variante.

**Status:** ⏸ Zurückgestellt – kein Fix in der Git-History identifiziert

## Hoch

### BUG-03 — Escape im Inline-Edit klappt die Sidebar auf

**Bereich:** Anbauflächen — Inline-Bearbeitung

**Schritte zur Reproduktion:**
1. Anbauflächen → Liste öffnen
2. Rechtsklick auf ein Beet → „Bearbeiten" (Zeile wechselt in Inline-Edit-Modus mit Textfeldern)
3. Escape drücken

**Erwartet:** Inline-Edit wird abgebrochen, Sidebar bleibt unverändert. (Dokumentiertes Kürzel: „Dialog schließen: Esc")

**Tatsächlich:** Sidebar klappt auf. Das Keyboard-Event von Escape steigt bis zum globalen Sidebar-Toggle-Handler hoch, anstatt vom Inline-Edit abgefangen zu werden.

**Vorgeschlagene Lösung:** Im Escape-Handler des Inline-Edits `event.stopPropagation()` aufrufen, damit das Event nicht zum Sidebar-Kürzel durchdringt.

**Status:** ⏸ Zurückgestellt – kein Fix in der Git-History identifiziert

### BUG-04 — Abbrechen im Kulturen-Bearbeiten-Modal verwirft Änderungen ohne Warnung

**Bereich:** Kulturen — Bearbeiten-Modal

**Schritte zur Reproduktion:**
1. Kulturen öffnen, eine Kultur auswählen
2. Bearbeiten-Button (Stift) klicken
3. Beliebiges Feld ändern (z. B. Namen tippen)
4. „Abbrechen" klicken

**Erwartet:** Warndialog erscheint, z. B. „Änderungen verwerfen?"

**Tatsächlich:** Modal schließt sich sofort, alle eingetippten Änderungen sind verloren.

**Vorgeschlagene Lösung:** Dirty-State erkennen und vor dem Schließen „Änderungen verwerfen?" abfragen.

**Status:** ✅ Behoben – `e055a782` *„fix: add discard confirmation and guard false dirty badge in CultureForm"*

---

### BUG-05 — „Ungespeicherte Änderungen" erscheint sofort beim Öffnen des Modals

**Bereich:** Kulturen — Bearbeiten-Modal

**Schritte zur Reproduktion:**
1. Kulturen öffnen, eine Kultur auswählen
2. Bearbeiten-Button klicken
3. Kein Feld anfassen – Badge unten links beobachten

**Erwartet:** Kein „Ungespeicherte Änderungen"-Hinweis, solange nichts geändert wurde.

**Tatsächlich:** Badge erscheint sofort beim Öffnen des Modals, noch vor jeder Eingabe.

**Vorgeschlagene Lösung:** Badge erst anzeigen, wenn mindestens ein Feldwert vom ursprünglich geladenen Wert abweicht.

**Status:** ✅ Behoben – `e055a782` *„fix: add discard confirmation and guard false dirty badge in CultureForm"*

### BUG-06 — Grafik-Ansicht zeigt Beet-Rechtecke ohne Beschriftung

**Bereich:** Anbauflächen — Grafik-Ansicht

**Schritte zur Reproduktion:**
1. Anbauflächen → Grafik öffnen
2. Einen Standort aufklappen

**Tatsächlich:** Alle Rechtecke sind leer. Kein Name, keine Nummer, kein Tooltip beim Hover. Die Grafik ist ohne gleichzeitige Nutzung der Listenansicht nicht interpretierbar.

**Vorgeschlagene Lösung:** Beet-Name (und optional Maße) in oder unter jedem Rechteck rendern, Schriftgröße an die verfügbare Fläche anpassen.

### BUG-11 — Versionsverlauf: neuester Eintrag hat keinen „Wiederherstellen"-Button

**Bereich:** Versionsverlauf-Modal

**Schritte zur Reproduktion:**
1. Beliebigen Eintrag löschen
2. Mehr → Versionsverlauf öffnen

**Erwartet:** Neuester Eintrag (die Löschung) ist als aktuell markiert UND bietet die Möglichkeit, den vorherigen Zustand wiederherzustellen.

**Tatsächlich:** Neuester Eintrag zeigt „Aktuelle Version"-Badge, aber keinen Wiederherstellen-Button. Ältere Einträge bieten projektweite Rollbacks, die auch andere Änderungen rückgängig machen.

**Vorgeschlagene Lösung:** „Letzte Änderung rückgängig"-Button beim neuesten Eintrag hinzufügen, oder direkt nach Lösch-Aktionen einen „Rückgängig"-Snackbar anzeigen.

**Status:** ✅ Behoben – `85642b91` *„fix: show version history dialog when only the current version exists"*
